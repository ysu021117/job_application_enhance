# Reference Consistency Check

Use this reference before delivering a revised resume or cover letter.

## Goal

Confirm that the final document is genuinely consistent with the user's supplied writing references or saved voice reference, not just generally well written.

## Required Check

Before final delivery, compare the revised draft against:

- the user's past professional writing samples
- or the saved `dynamic-reference/voice-reference.md` if one exists

Do not skip this step when reference material is available.

## What To Compare

Check for alignment on:

- formality
- directness
- warmth
- self-promotion level
- sentence rhythm
- vocabulary style
- tendency toward brevity or expansion

## Failure Conditions

Treat the draft as not ready if it:

- sounds more corporate than the user's references
- sounds more enthusiastic than the user's references
- sounds flatter or more generic than the user's references
- uses polished but non-native phrasing that the user would be unlikely to use
- copies reference wording too literally instead of matching the underlying style

## Pass Standard

The final draft should feel like:

- the same person wrote it for a hiring context
- a cleaner and stronger version of the user's own professional voice
- a document that aligns with the references without becoming a copy of them

## If The Draft Fails

If the consistency check fails:

1. identify which voice dimensions drifted
2. revise the draft again
3. rerun the consistency check before delivery

## Priority Rule

Reference consistency is important, but it does not override:

1. factual accuracy
2. explicit user instructions
3. core resume or cover-letter conventions
