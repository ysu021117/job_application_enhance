# Source Extraction

Use this reference when consolidating multiple resume versions and supporting materials into reusable candidate evidence.

## Goal

Extract what the candidate has actually done, learned, published, built, or achieved, without anchoring the reference set to any single source wording.

## Source Types

Handle source materials such as:

- resume versions
- CVs
- portfolio pages or project lists
- articles, essays, or published writing
- transcripts
- bios or profiles
- prior application documents

## Extraction Rules

- Extract facts, responsibilities, outcomes, tools, topics, domains, and quantified evidence.
- Preserve numbers exactly when the source supports them.
- Normalize wording across versions into one evidence record per activity, achievement, or skill area.
- Keep track of source support when multiple versions describe the same work differently.
- Prefer the strongest accurate quantified version when several sources describe the same result.
- Do not collapse distinct experiences into one if they are truly separate.

## What To Preserve

Preserve completely:

- metrics
- dates
- scale indicators
- named outputs such as publications, projects, products, or programs
- tools, methods, subject areas, and technical stacks
- academic performance signals that are relevant and supported

## What To Avoid

- copying polished bullet language verbatim into the reference
- dropping quantified evidence because one source version is more concise
- treating one resume version as canonical if the set of files clearly contains more complete evidence
- inventing connective facts between partial sources

## Reference Files To Create

Create at least:

- `reference/master-experience-reference.md`

Create additional files when useful:

- `reference/education-and-transcript-reference.md`
- `reference/portfolio-reference.md`
- `reference/source-reconciliation-notes.md`

## Recommended Reference Shape

For each normalized experience entry, capture:

- area or role
- normalized summary of what was done
- quantified evidence
- supporting skills, tools, or domains
- source note if needed for later verification

Start from `assets/templates/master-experience-reference-template.md` when useful.
